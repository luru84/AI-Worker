# Data Dictionary

- `intake`: 要件が固まる前の相談メモ
- `assistant-agent`: ユーザーとの窓口役。整理と割当起案を行う
- `goal`: その作業で達成したいこと
- `plan`: 親 Codex の段取り
- `task`: 子 Codex への割当
- `security review`: secrets, PII, prompt injection, permission を確認するレビュー
- `publish step`: GitHub への反映段階
- `target repo`: 実装や GitHub 操作を行う実リポジトリ
- `decision`: 複数作業をまたいで参照する重要判断
